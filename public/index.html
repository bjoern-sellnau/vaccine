Configurable shims for modular JavaScript

Vaccine generates shims to make CommonJS (Node.js) or AMD modules work in the browser. Configurability makes the shims small by only including the required functionality. A small size is especially important for libraries.

To use Vaccine, first set the options in the right panel. The shim will instantly adjust to changes. Use the diff button to see the differences between two different configurations.

Next, copy vaccine.js and build.sh to your project's root directory. When you are ready to build, run sh build.sh > my_built_file.js. Note that all source code, and only source code, must be in a separate directory for the tiny build script to work.

Use the UMD template for single file projects—no build required.

For development, the vaccine_dev.js version can be placed before script tags for all the modules (in any order). Call requireDev() to get the project's global.

The name "vaccine" comes from the analogy that the shims are injected into the code like the virus used in a vaccine, and that they are a temporary treatment until the Harmony modules "cure" arrives.
